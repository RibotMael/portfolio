Présentation du projet

Le projet Bataille Navale est une création de RQM Studio, une équipe de trois développeurs passionnés. L'objectif est de moderniser ce classique pour proposer une expérience multijoueur riche, accessible sur tout type d'ordinateur et prochainement sur mobiles.

Le jeu propose des modes variés : du 1v1 classique aux combats en équipe (2v2, 3v3, 4v4), jusqu'au mode Battle Royale réunissant plusieurs flottes simultanément.

Le système de jeu intègre deux variantes de règles : la version française (5 bateaux) et la version belge (10 bateaux), avec un rythme dynamique imposant 7 secondes de réflexion par tour.

Stack Technique & Fonctions

L'architecture repose sur une séparation claire entre l'interface utilisateur et la logique métier. Le projet est structuré en deux dépôts distincts : Battleship-ui et Battleship-api.

Vue.js — Frontend
Node.js — Runtime
Express — API REST
MySQL — Base de données
Chiffrement — Sécurité
Responsive — UI/UX
GitHub — Versionning

WebSocket — Temps réel

Périmètre & Objectifs

Le projet vise à offrir une plateforme stable avec une interface intuitive. Les joueurs peuvent gérer leur profil, une liste d'amis et débloquer des récompenses cosmétiques.

Temps de jeu limité (7s)

Développement interne sur une durée estimée à 2 ans, incluant des phases de bêta fermée et de tests intensifs avant le déploiement officiel.

Équipe RQM Studio

L'ensemble du développement est réalisé en interne par les trois membres fondateurs de RQM Studio dans le cadre du cursus BTS SIO, chacun utilisant un langage différent.

Le public cible : des joueurs occasionnels cherchant une partie rapide aux fans de stratégie désirant une version modernisée avec un aspect compétitif fort.
